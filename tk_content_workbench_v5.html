3 步上手
第一次交付给客户或团队成员时，按这个顺序使用最稳。
1
配置模型与 API Key
填写使用者自己的 OpenAI Key，选择模型后即可开始生成。Key 默认只在当前浏览器环境中使用。
2
选择产品并生成内容
从 SKU 下拉中选品，系统会自动带出产品知识卡、参考图片和更垂直的表达方向。
3
扩展产品并同步资料
在“扩展产品”里上传新产品和图片，再导出 JSON 或同步到 GitHub，方便团队共享。

GitHub 仓库同步

会同步三类数据到仓库：`custom-products.json`、`generation-history.json`、以及扩展产品上传的图片文件。Token 需要有仓库 contents 写权限。
还没有执行 GitHub 同步。仓库配置会保存在当前浏览器，方便下次继续同步。
如果你要把工具交给别人，可以把 `html + assets 文件夹 + 导出的 json` 一起发给对方。对方导入后就能拿到扩展产品库和生成历史。

你的 GitHub 数据仓库接收

推荐做法：这里填一个超低成本的 Cloudflare Worker 接收地址。前端会把扩展产品、文案历史、AI 图片库一并打包 POST 过去，再由 Worker 写入你自己的 GitHub 数据仓库。
还没有执行云端接收同步。配置会保存在当前浏览器，方便用户下次继续提交。
